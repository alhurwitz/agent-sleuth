# agent-sleuth v0.1.0 — Initial release

> **Status:** Draft. Edit before tagging. Use the body of this file as the GitHub release notes when you push the `v0.1.0` tag.

## TL;DR

`agent-sleuth` is a plug-and-play agentic search library for Python. Bring your own LLM, point it at one or more **backends** (local files, code, web search, vector stores, custom), and call `agent.aask("...")` to stream a fully-cited answer through a typed event stream. Use it directly as a Python SDK, behind any of eight popular agent-framework adapters, or via the `sleuth-mcp` server.

```bash
pip install agent-sleuth[anthropic,langchain]
```

```python
from sleuth import Sleuth
from sleuth.backends import LocalFiles, WebBackend
from sleuth.llm import Anthropic

agent = Sleuth(
    llm=Anthropic(model="claude-sonnet-4-6"),
    backends=[
        WebBackend(provider="tavily", api_key=...),
        LocalFiles(path="./docs"),
    ],
)

async for event in agent.aask("How does our auth flow handle refresh tokens?"):
    print(event.type, event)
```

## Highlights

- **One unified Q&A pipeline** — Router → Planner → Executor → Synthesizer, with a single typed event stream as the primary surface (`RouteEvent`, `PlanEvent`, `SearchEvent`, `FetchEvent`, `ThinkingEvent`, `TokenEvent`, `CitationEvent`, `CacheHitEvent`, `DoneEvent`).
- **PageIndex-style local search by default** — hierarchical tree-of-contents per document, no embeddings infrastructure required. Vector-RAG is opt-in via `VectorStoreRAG` for teams that already have an index.
- **BYOK LLM, BYO backends** — `LLMClient` and `Backend` are small protocols; the package never imports a model SDK as a hard dependency. Anthropic and OpenAI shims live behind extras.
- **Eight framework adapters out of the box** — LangChain, LangGraph, LlamaIndex, OpenAI Agents SDK, Claude Agent SDK, Pydantic AI, CrewAI, AutoGen.
- **MCP server** — `sleuth-mcp` exposes `search` and `summarize` over stdio + HTTP; backends configured server-side via TOML.
- **Streaming, cited, structured** — every answer carries citations to real sources; pass `schema=YourPydanticModel` to get typed structured output without changing the rest of the pipeline.
- **Cache-hit replay** — cached query results replay through the same event stream prefixed with `CacheHitEvent`, so consumers never need a special cache-aware code path.

## What's in the box

### Engine

- **Heuristic Router** — classifies depth (`auto` / `fast` / `deep`) without an LLM call.
- **Planner + reflect loop** (`depth="deep"`) — decomposes multi-part queries, runs sub-searches in parallel, reflects on results, and bails out when the planner says "done" or `max_iterations` is exhausted.
- **Speculative prefetch** — backend search starts on the first sub-query while the planner is still streaming the rest, hiding planner latency behind search latency.
- **Streaming Synthesizer** — emits tokens as the LLM generates them, citations resolve in parallel.
- **Per-backend timeouts** — `asyncio.wait_for` wrap; defaults 8 s web / 4 s local, configurable per backend via duck-typed `timeout_s`. Failures surface as `SearchEvent(error=...)` and the run continues with whatever else returned.

### Backends

| Backend | Description |
| --- | --- |
| `LocalFiles` | PageIndex-style tree-of-contents (markdown, PDF via `pymupdf`, HTML, code via tree-sitter); persists per-corpus at `<dir>/.sleuth/index/`. |
| `CodeSearch` | Two-phase retrieval: ripgrep for lexical hits → tree-sitter expand to enclosing function/class. SQLite symbol index. Optional embedding re-rank (`code-embed` extra). |
| `WebBackend` (factory) | One unified provider entry point, plus `TavilyBackend`, `ExaBackend`, `BraveBackend`, `SerpAPIBackend` for power users. Per-domain rate limit + exponential backoff. Optional `fetch=True` mode (parallel-fetches top-N pages, chunks via `trafilatura` + `tiktoken`). |
| `VectorStoreRAG` | Opt-in adapter wrapping a `VectorStore` protocol. Vendor adapters for Pinecone, Qdrant, Chroma, Weaviate. |
| Custom | Implement `Backend` (one async `search(query, k) -> list[Chunk]`) and pass it to `Sleuth(backends=[...])`. `BackendTestKit` validates protocol compliance. |

### Memory

- `SqliteCache` (default) at `~/.sleuth/cache/{query,fetch,plan}.sqlite` with per-namespace TTLs (10 min / 24 h / 1 h).
- `IndexCache` (per-corpus) lives next to the data at `<corpus>/.sleuth/index/` — never in user home.
- `SemanticCache` (opt-in, `semantic` extra) — fastembed BGE-small embedder, 0.92 cosine threshold, 10 min window. Pluggable `Embedder` protocol for swapping in any embedding provider.
- `Session` ring buffer with `save(path)` / `Session.load(path)` (classmethod) / `await flush()` for multi-turn coherence.

### LLM shims (BYOK)

- `sleuth.llm.Anthropic` (`anthropic` extra) — adapts the Anthropic SDK stream to `LLMClient`. `supports_reasoning=True` for Claude extended-thinking models when `thinking=True`.
- `sleuth.llm.OpenAI` (`openai` extra) — adapts the OpenAI SDK stream to `LLMClient`. `supports_reasoning=True` for o-series models. Both shims lazy-import the SDK so missing extras only error at instantiation.
- `sleuth.llm.StubLLM` — deterministic test double; cycles scripted responses per call.

### Frontends

| Frontend | Module | Surface |
| --- | --- | --- |
| Native Python SDK | `sleuth` | `Sleuth.ask` / `aask` / `summarize` / `asummarize` |
| LangChain (Tier 1) | `sleuth.langchain` | `SleuthTool`, `SleuthRetriever`, `SleuthCallbackHandler` |
| Claude Agent SDK (Tier 1) | `sleuth.claude_agent` | `SleuthClaudeTool` (event → progress block mapping) |
| LangGraph | `sleuth.langgraph` | `make_sleuth_node` factory |
| LlamaIndex | `sleuth.llamaindex` | `SleuthQueryEngine`, `SleuthRetriever` |
| OpenAI Agents SDK | `sleuth.openai_agents` | `make_sleuth_function_tool` |
| Pydantic AI | `sleuth.pydantic_ai` | `SleuthInput`, `make_sleuth_tool` |
| CrewAI | `sleuth.crewai` | `SleuthCrewAITool` (with `on_event` callback) |
| AutoGen | `sleuth.autogen` | `make_sleuth_autogen_tool`, `register_sleuth_tool` |
| MCP server | `sleuth-mcp` | `search` / `summarize` tools over stdio + HTTP |

## Install

```bash
# Core (no LLM/framework deps)
pip install agent-sleuth

# Add an LLM
pip install 'agent-sleuth[anthropic]'   # Anthropic shim
pip install 'agent-sleuth[openai]'      # OpenAI shim

# Add a framework
pip install 'agent-sleuth[langchain]'
pip install 'agent-sleuth[claude-agent]'
pip install 'agent-sleuth[langgraph]'
pip install 'agent-sleuth[llamaindex]'
pip install 'agent-sleuth[openai-agents]'
pip install 'agent-sleuth[pydantic-ai]'
pip install 'agent-sleuth[crewai]'
pip install 'agent-sleuth[autogen]'

# Backends
pip install 'agent-sleuth[localfiles]'           # LocalFiles (pymupdf for PDFs)
pip install 'agent-sleuth[code]'                 # CodeSearch (tree-sitter)
pip install 'agent-sleuth[code-embed]'           # CodeSearch optional embed re-rank
pip install 'agent-sleuth[exa]'                  # Exa web search
pip install 'agent-sleuth[web-fetch]'            # fetch=True mode (trafilatura, tiktoken)
pip install 'agent-sleuth[pinecone]'             # VectorStoreRAG vendor
pip install 'agent-sleuth[qdrant]'
pip install 'agent-sleuth[chroma]'
pip install 'agent-sleuth[weaviate]'

# Memory
pip install 'agent-sleuth[semantic]'             # SemanticCache (fastembed)

# MCP server
pip install 'agent-sleuth[mcp]'
sleuth-mcp --transport stdio
sleuth-mcp --transport http --host 127.0.0.1 --port 8765
```

CLI (after the MCP install):

```bash
sleuth-mcp --config ~/.config/sleuth/mcp.toml --transport stdio
```

Example `mcp.toml`:

```toml
[llm]
name = "anthropic:claude-sonnet-4-6"

[[backends]]
type = "web"
provider = "tavily"
api_key_env = "TAVILY_API_KEY"

[[backends]]
type = "localfiles"
path = "/var/data/docs"
```

## Known limitations

- **Schema-typed results bypass the cache.** When `Sleuth.aask(..., schema=YourModel)` is used, results are not currently round-trippable through JSON serialization, so cache writes are skipped on those calls. v0.2 follow-up.
- **Cache-hit replay is text + citations only.** The replayed `TokenEvent` emits the full cached text in a single chunk rather than reconstructing the original token-by-token stream. The `Result` is identical; consumer behavior is unchanged.
- **`SemanticCache` requires the `semantic` extra at runtime.** Importing `sleuth.memory.semantic` without the extra succeeds; calling `FastembedEmbedder.embed()` or `SemanticCache.lookup()` raises a clear `ImportError` pointing at the extra.
- **Default `SqliteCache` writes to `~/.sleuth/cache/`.** Pass `cache=None` for stateless / test environments, or `cache=SqliteCache(base_path=...)` to override.
- **`pip-audit` ignores `GHSA-58qw-9mgm-455v`.** `pip <=26.0.1` has an unfixed zip/tar interpretation conflict (CVE-2026-3219); the ignore is documented in `ci.yml` and will be dropped when `pip` ships a fix.
- **AutoGen extra resolves to `autogen-agentchat`.** The plan referenced the `pyautogen` package; PyPI's `pyautogen>=0.3` proxies to `autogen-agentchat>=0.10`. The adapter targets the new v0.4+ API (`model_client=` instead of `llm_config=`).

## Out of scope for v0.1.0 (planned v0.2+)

- Vector-backed `Session` for multi-thousand-turn conversations
- Built-in evaluation harness (correctness, faithfulness, citation accuracy)
- Long-term cross-session vector memory
- Fine-tuning the navigator LLM on user-specific corpora
- Multi-modal (image / audio / video) document indexing
- OpenTelemetry adapter for engine traces / metrics
- Round-trippable schema results in the query cache

## Acknowledgements

The hierarchical local-files backend builds on the [PageIndex](https://github.com/VectifyAI/PageIndex) technique (VectifyAI, 2024) — reasoning-based document indexing that replaces vector chunking with an LLM-navigated table of contents.
